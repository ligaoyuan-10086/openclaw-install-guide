# OpenClaw 安装指南项目配置

## 项目说明

本项目用于记录 OpenClaw 的安装过程、问题排查和解决方案。

## 文档管理流程

### 使用流程

```
遇到新问题
    ↓
📝 立即记录到 shrimp-diary.md
   - 记录日期：YYYY年MM月DD日
   - 记录环境信息
   - 记录错误信息（截图/文本）
   - 记录所有尝试过程（包括失败的）
   - 记录最终解决方案
   - 总结经验教训
    ↓
✅ 问题解决后
    ↓
📚 整理到 troubleshooting.md
   - 按问题类型分类
   - 标准化格式
   - 添加原因分析
   - 提供多种解决方案
   - 添加性能对比（如适用）
    ↓
🔥 如果是高频问题
    ↓
⚡ 提炼到 installation-faq.md
   - 问答式格式
   - 简洁明了
   - 快速定位
   - 提供快速解决方案
```

### 三者区别

| 维度 | shrimp-diary.md | troubleshooting.md | installation-faq.md |
|------|----------------|-------------------|-------------------|
| **定位** | 个人记录 | 技术文档 | 快速查询 |
| **读者** | 自己/同行 | 遇到问题的用户 | 快速查找的用户 |
| **风格** | 随意、真实 | 专业、系统 | 简洁、直接 |
| **内容** | 完整过程 | 标准方案 | 核心答案 |
| **更新** | 实时记录 | 定期整理 | 提炼高频 |
| **价值** | 保留思考 | 系统知识 | 快速解决 |

## Claude 工作指引

### 记录新问题时

1. **立即在 shrimp-diary.md 中记录**
   - 使用日期标题：`## YYYY年MM月DD日`
   - 完整记录环境信息（系统、版本、路径等）
   - 将错误信息放入代码块中
   - 记录所有尝试的命令和输出（包括失败的）
   - 记录最终解决方案
   - 添加经验总结

2. **问题解决后整理到 troubleshooting.md**
   - 按问题类型分类（安装问题、配置问题、运行问题等）
   - 使用标准格式：
     - 问题描述
     - 环境信息
     - 错误信息
     - 原因分析
     - 解决方案（可以有多个）
     - 相关参考

3. **高频问题提炼到 installation-faq.md**
   - 使用问答格式
   - 问题要具体明确
   - 答案要简洁直接
   - 提供快速解决命令

### 文档更新原则

- **实时性**：遇到问题立即记录到 shrimp-diary.md
- **完整性**：保留所有尝试过程，包括失败的
- **准确性**：代码块中的错误信息要完整准确
- **可读性**：使用 Markdown 格式，代码块、标题、列表等
- **渐进式**：从日记 → 故障排查 → FAQ，逐步提炼

### 推送规则

- 每次更新文档后，自动推送到 GitHub 和 Gitee
- 提交信息格式：`更新：[文档名] - [简短描述]`
- 同时推送到两个远程仓库
